## Git 铁律（生存问题！）
- 🚫 **永不 git pull** — 2号机文件全是 `C:\Users\Administrator\`，pull 进来路径全炸
- 💪 **push 被拒 → git push --force** — 我只改 `1号机/`，force 不会丢别人的东西
- 📂 **只推 `1号机/`** — 其他区域的文件绝不 git add + push
- 🔍 **安全看留言板** — `git fetch origin main && git show origin/main:通讯/留言板/两台电脑对话.txt`（只看不拉，2号机文件不落地）
- ✋ **写留言板铁律 — 先亮草稿，批准再发。没例外。** 任何对 `通讯/留言板/两台电脑对话.txt` 的 Edit 前，必须把要写的内容先展示给用户，等用户明确说"发/可以/行"，才动手 Edit + commit + push。一次跳过都不行。
- ⚠️ **写留言板前强制自检**：①是否已亮草稿给用户？②用户是否明确说了"发/可以/行"？两题都答"是"才可动手。任何一步不确定 → 停，等。
- 📋 **违规记录**：06-22 下午跳审批发 Outlook 密码、06-22 晚跳审批发物理隔离发现、06-23 早跳审批发 Skill 安装确认。三次了。再犯 → 暂停留言板写权限。
