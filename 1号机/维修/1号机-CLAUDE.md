- ⚠️ **系统改造前必须遵循「6步改造流程」**：涉及脚本/代码/配置/数据库/API/定时任务的任何改动，必须先出改造报告存档到 `数据库改造/`，用户批准后再动手。不得边改边想、改完才报。

## 系统改造 6 步流程

任何系统级改动（脚本/代码/配置/API/定时任务/数据库）必须按以下流程，禁止跳过：

| 步 | 动作 | 说明 |
|:--:|------|------|
| 1 | **说明现状问题** | 当前什么坏了、什么场景触发、影响范围 |
| 2 | **说明改动措施 + 隐患** | 将要改什么文件、具体改法、可能的副作用和风险 |
| 3 | **说明改动后效果** | 改完后的预期行为、对比改前 |
| 4 | **报告存档到 `数据库改造/`** | 生成 `改造报告-{主题}-{日期}.md` |
| 5 | **报请用户确认** | 列出将要改动的文件清单，等批准 |
| 6 | **改动完记录结果** | 在报告末尾追加：实际改动内容 + 验证结果 + 有无意外 |

> ⚠️ 第 5 步用户没批准之前，只出报告不执行。第 6 步改完必须补记录。

## 访问网页的规则
- **不要使用 WebFetch 工具访问网站**——在当前网络/安全策略下经常被拦截
- 统一使用 `curl` 命令拉取网页或 API 内容:
  - 普通网页:`curl -sL "URL" -o /tmp/page.html` 然后 Read,或者直接 `curl -sL "URL" | head -200`
  - GitHub 仓库:优先用 `gh repo view owner/name`、`gh api repos/owner/name/...`
  - JSON API:`curl -sL "URL" | jq` 便于查看
- 仅当 `curl` 也无法获取时再考虑 WebSearch 或其他方案

## 主动解决问题规则

**第一次失败不要立刻回报"做不到",先穷尽办法再说。** 像 Claude Code / OpenClaw 那样主动想办法。

### 缺 skill / 缺工具时
按顺序自动尝试,找不到再问用户:
1. 本地 `SKILL清单/本地Skills清单.md` 和 `/mnt/skills/` 检索
2. `gh search repos` 搜 GitHub,关键词如 `claude skill <主题>`、`<功能> mcp server`
3. 翻 `awesome-claude-skills`、`awesome-mcp`、`awesome-claude-code` 等聚合仓库
4. 找到就装到 `SKILL清单/` 下,同步更新 `本地Skills清单.md`
5. 实在找不到,列 2–3 个候选让用户选,不要空手回

### GitHub 网络失败时:动态查找镜像源
**不要使用任何写死的镜像源列表**——镜像源换得很快,半年前能用的半年后就挂了。流程如下:

1. **先重试 1 次**:可能只是网络抖动。
2. **检查本地缓存的源**:读 `SKILL清单/本地Skills清单.md` 的"可用镜像源"段落,如果有近期(< 30 天)验证过的源,直接用。
3. **实时搜索当前可用源**:用 WebSearch 查 `github 镜像 可用 <当前年月>`,从搜索结果里挑 1–2 个看起来最新的镜像域名(优先博客发布时间近、明确写"实测可用"的)。
4. **逐个 curl 探测**:对挑出的候选,先 `curl -sI --max-time 10 <镜像URL>` 看返回状态,200/302 才算活。
5. **用活的源完成任务**,然后**把这个源写进 `本地Skills清单.md` 的"可用镜像源"段落**,记录:域名、用途(git clone / raw / release)、验证日期。下次先用本地缓存。
6. **三轮搜索都找不到活源** → 回报用户,列出已试过的域名和具体失败原因。

**包管理器同理**:npm / pip / cargo 等装包失败时,先查清华、阿里、npmmirror 这些大站(它们相对长期稳定,可以写死),不行再走上面的动态搜索。

### 命令报错自愈
- `command not found` → 自查 + `apt-get install` / `pip install` / `npm i -g`,装完重试
- 权限问题 → 试 `sudo` 或改写到 `/home/claude`
- Python 包冲突 → `pip install --break-system-packages`
- 装包失败 → 走上面"动态查找镜像源"的流程

### 失败回报格式
真的搞不定再回报,必须包含:① 试过哪些路径 ② 具体失败原因 ③ 至少 1 个下一步方向(要授权 / 要凭据 / 让用户选方案)。
